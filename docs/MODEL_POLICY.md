# CAPA model policy

CAPA separates reasoning from execution.

Use a high-reasoning model for:

- understanding ambiguous user intent;
- acting as Product Owner;
- shaping backlog;
- creating PBIs;
- decomposing tasks;
- defining acceptance criteria;
- deciding scope and risk.

Use a lower-cost execution model for:

- editing files inside approved scope;
- running commands;
- registering evidence;
- registering tests;
- registering reviews;
- completing one CAPA transition.

Backlog tasks should include:

```text
- title;
- description;
- acceptance criteria;
- recommended execution model;
- scope notes;
- test expectation;
- risk notes when needed.
```

The PO behavior turns conversation into this structure before execution starts.
